# Intent Kit vs Spec Kit

GitHub Spec Kit proves the market wants executable development artifacts. Intent Kit accepts that product shape but changes the center of gravity.

## Spec Kit center
- Constitution
- Specification
- Plan
- Tasks
- Implementation

## Intent Kit center
- Business intent
- User/customer intent
- Repo context
- Architecture intent
- Implementation plan
- Tasks
- Verification
- Evidence
- Impact
- Learning loop

## Coexistence model

Intent Kit can sit above Spec Kit:

- Intent Kit captures enterprise intent, repo context, evidence, and impact.
- Spec Kit can be used inside one implementation phase to create a detailed feature specification.
- Claude Code, Copilot, Cursor, Codex, and Gemini can all consume the artifacts.

## Differentiator

Spec Kit makes the spec executable. Intent Kit makes delivery accountable.
