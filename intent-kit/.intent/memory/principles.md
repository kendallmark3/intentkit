# IDE Principles

1. Business intent is the source of truth.
2. The repo is evidence, not a black box.
3. Agents must read before they write.
4. Every implementation needs acceptance evidence.
5. Every delivery loop should improve future delivery.
6. Architecture rules are constraints, not suggestions.
7. Human decisions must be captured in the decision log.
8. Do not optimize for generated code; optimize for verified outcomes.
