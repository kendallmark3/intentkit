# Brownfield Onboarding Playbook

Use this before implementing in an existing repo.

1. Clone and run the repo locally.
2. Identify package manager, build tool, test commands, lint commands, and deployment assumptions.
3. Map important folders and architectural layers.
4. Identify existing patterns for UI, API, state, validation, errors, logging, tests, and auth.
5. Create or update `.intent/memory/architecture-rules.md` with repo-specific constraints.
6. Create the first feature intent.
7. Do not let the agent code until repo-context.md has evidence.
